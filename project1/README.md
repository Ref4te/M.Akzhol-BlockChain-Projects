## 1. Подготовка рабочего окружения

Откройте терминал и перейдите в директорию проекта:

```bash
cd projects/project1
npm install
```

**Что происходит:**
- `npm install` устанавливает необходимые зависимости (`ethers`, `dotenv`) в папку `node_modules`
- Эти библиотеки понадобятся для взаимодействия с блокчейном

---

## 2. Запуск локальной сети (Anvil)

Откройте **новое окно терминала** и запустите локальный блокчейн:

```bash
anvil
```

**После запуска вы увидите:**
- ✅ 10 тестовых аккаунтов с балансом 10000 ETH каждый
- 🔑 Приватные ключи для каждого аккаунта
- 🌐 RPC URL: `http://127.0.0.1:8545`

> ⚠️ **Важно**: Это окно должно оставаться открытым всё время работы — здесь работает ваш локальный блокчейн!

---

## 3. Настройка переменных окружения (.env)

Вернитесь в **первое окно терминала** и создайте файл `.env`:

```bash
touch .env
```

Откройте файл и добавьте следующие данные:

```env
PRIVATE_KEY=0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80
RPC_URL="http://127.0.0.1:8545"
```

> 🔐 **Для тестовой сети**: Можно использовать RPC тестовой сети (например, Goerli или Sepolia) и приватный ключ от MetaMask

**Назначение файла `.env`:**
- Безопасное хранение приватного ключа
- Конфигурация подключения к блокчейну

---

## 4. Компиляция смарт-контракта

Скомпилируйте ваш контракт с помощью `solcjs`:

```bash
solcjs --bin --abi --include-path node_modules/ --base-path . Note.sol
```

**Результат компиляции:**
- 📄 `Note_sol_Note.abi` — интерфейс контракта для JavaScript
- 🔤 `Note_sol_Note.bin` — байткод для загрузки в блокчейн

---

## 5. Деплой контракта

Запустите скрипт развертывания:

```bash
node deploy.js
```
Скрипт подключится к локальной сети Anvil, подпишет транзакцию вашим Private Key и отправит контракт в блокчейн. После успешного деплоя в терминале появится адрес задеплоенного смарт-контракта.
